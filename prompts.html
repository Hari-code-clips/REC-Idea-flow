Room 02 · Prompt Library

From pain point
to a runnable pilot prompt.

Five prompts, designed to be run in order. Each one refines the thinking from the last — from discovery and diagnosis, through ideation, to a brief and finally a platform-ready build prompt. Use them with any frontier LLM.

How the flow works

Start with Widen to map personas and pains, then Diagnose to find root causes. Ideate converges on a pilot concept. Brief turns it into a one-pager. Build writes the product requirements prompt you ship to a build platform (Lovable, v0, Google AI Studio, etc.).

Tip: replace bracketed tokens like [selected challenge] with the challenge you picked from Room 01.

STEP 02 / ROOT-CAUSE

Diagnose

Pick the top pains and run a structured Five Whys.

five-whys test-plan privacy
Let's pick the top [#] pains for this challenge. For these listed top pains, run a Five Whys. Propose 3 root-cause hypotheses and the disproof evidence for each. Specify the minimum data cut & owners to pull. Output a root-cause map, test plan, and privacy constraints.
REPLACE [#] WITH COUNT
STEP 03 / CONVERGE

Ideate

Cluster AI-driven ideas. Score. Recommend a pilot.

process analytics/ML automation impact × feasibility × confidence × TTV
Generate and Cluster possible AI driven ideas into 3 Options: 1. Process (policy, ways of working) 2. Analytics / ML (forecast, optimise, recommend) 3. Automation (CV, RAG / Co-Pilot, tasking) Score each on Impact × Feasibility × Confidence × Time-to-Value. Recommend one pilot with the smallest integration surface.
CONVERGE TO ONE PILOT
STEP 04 / CONDENSE

Brief

Turn the chosen pilot into a one-page pilot brief.

one-pager metrics & baselines guardrails
For option [#], create a one-page pilot brief including: Target user(s), problem statement, success metrics & baselines, target uplift, key flow (5–7 steps), screens / components, sample UI copy, representative sample data, integration points, and relevant guardrails (GDPR / PCI, domain-specific regulation boundaries, bias tests, fallback behaviour).
ONE PAGE · CONCRETE
STEP 05 / SHIP

Build

Generate the platform-ready product requirements prompt.

PRD-as-prompt screens + components no PII
SELECT PLATFORM
You are a product design expert. Using only the brief above, write a single [platform] product requirements prompt that includes: Product name + one-liner description (actions, process, capabilities), who it's for, screens + key components, brand colors, main user flow, sample data, concise headlines / CTAs, UI instructions, success metric card, constraints (no PII).
